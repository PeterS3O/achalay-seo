Bug técnico verificable

72 URLs paginadas canonicalizadas a página 1.

TLW usa paginación WordPress correcta (/tecnologia/page/2/, etc.) pero muchas de estas páginas paginadas declaran canonical hacia página 1. Google desde 2011 recomienda explícitamente NO hacer esto — cada página paginada debe ser auto-referencial. El bug hace que Google deduplique todo el listado en una sola URL, perdiendo autoridad y crawl budget.

Estado actual
Ejemplo URL paginada:
/tecnologia/page/60/
Response:
200 OK · HTML válido con 7 artículos
Canonical actual:
<link rel="canonical" href="/tecnologia/">
Resultado:
Google descarta esta URL, no consolida los artículos de página 60
Corrección recomendada
Cambiar canonical a self-referencial:
<link rel="canonical"
href="/tecnologia/page/60/">
Meta title distinto:
<title>Tecnología · Página 60</title>
Robots meta:
<meta name="robots" content="index,follow">
Resultado:
Google indexa cada página, distribuye autoridad, crawl budget se usa eficientemente
Impacto medido: 72 páginas paginadas están descartadas por Google por este bug. Categorías afectadas incluyen /tecnologia/ (páginas 2-60), /innovacion/, y varias más. Cada una lista 7 artículos que Google no puede descubrir eficientemente. Es P2 del roadmap — es un fix de una línea en functions.php o en el plugin de SEO (Yoast/RankMath/AIOSEO).

Impacto en Core Web Vitals

Response time promedio: 5,22 segundos. Google penaliza esto directamente.

A diferencia del blog editorial promedio, TLW tiene un problema serio de performance del servidor. El TTFB (Time To First Byte) medido por Screaming Frog es 5,22s promedio, con 491 URLs (24%) que tardan más de 10 segundos en responder. Core Web Vitals (LCP en particular) es factor de ranking desde 2021 y penaliza directamente.

Diagnóstico: combinación de servidor sub-dimensionado + HTML inflado (86% de páginas exceden 300 KB) + plugins WordPress que corren PHP en cada request sin cachear. Solución típica: cache de página completa (WP Rocket, LiteSpeed Cache), CDN (Cloudflare full page cache), optimización de queries DB, y reducción de scripts inline.
